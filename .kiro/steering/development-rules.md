# 开发规则强制执行

本文件确保所有 spec 任务执行时严格遵守项目开发规则。

## 强制规则

### 1. 包管理器规范
- ✅ **必须使用 pnpm**，禁止使用 npm
- 所有安装命令使用 `pnpm install`
- 所有运行命令使用 `pnpm run <script>`
- 所有添加依赖使用 `pnpm add <package>`

### 2. 服务依赖规范
- ✅ 本地代码（前端/后端）直接在本地运行
- ✅ 数据库、Redis、消息队列等通过 docker-compose 启动
- ❌ 禁止在本地直接安装 PostgreSQL、MySQL、Redis 等服务

### 3. 代码清理规范
- 任务完成后必须清理无用代码和文件
- 删除所有临时文件、测试文件、备份文件
- 删除冗余的脚本和配置
- **最多只保留一个 MD 说明文档**
- 保持项目结构清晰简洁

### 4. 禁用命令
- ❌ **禁止使用 `cat << 'EOF'` 命令**
- 该命令会在 terminal 执行过程中出错，导致执行崩溃
- 使用 fsWrite 或 strReplace 工具代替

## 执行检查清单

在执行任何任务前，确认：
- [ ] 使用 pnpm 而非 npm
- [ ] 第三方服务通过 Docker 启动
- [ ] 不使用 cat << 'EOF' 命令
- [ ] 任务完成后清理无用文件

在任务完成后，确认：
- [ ] 删除了所有临时文件
- [ ] 删除了所有备份文件
- [ ] 合并了多余的文档文件
- [ ] 项目结构清晰简洁
